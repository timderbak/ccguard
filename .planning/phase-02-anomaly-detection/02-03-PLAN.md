<objective>
Wire the math from plan 02-02 into a hands-free background loop: a single `tick()` function that iterates every known machine, computes all 4 metric series + baselines, persists the baselines via UPSERT, and emits a `FindingRecord` (severity=warn) when the latest point exceeds `mean + 3σ` AND `baseline_ready=True`. Same-day re-emission for an existing `(machine_id, rule_id)` is suppressed at the service layer (per RESEARCH recommendation — no DB UNIQUE constraint). The tick is driven by APScheduler embedded in the FastAPI lifespan with a 30-second first-tick (dev visibility) and an hourly interval thereafter. A test-only env guard `CCGUARD_DISABLE_SCHEDULER=1` keeps `TestClient` from hanging on the scheduler thread.

Purpose: Detection has to happen without operator intervention. The scheduler is the only piece of Phase 2 that runs autonomously; everything else is a render of its output.

Output: New `anomaly_service.py`, new `scheduler.py`, lifespan-integration in `main.py`, conftest fixture forcing `CCGUARD_DISABLE_SCHEDULER=1` for the whole test suite, ≥10 unit tests of tick() with the scheduler bypassed (call tick directly), and the apscheduler dep added to pyproject.toml (pinned `>=3.10,<4` — NOT 4.x alpha).
</objective>

Test 5: payload fields all present.
