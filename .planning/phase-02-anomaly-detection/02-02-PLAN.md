  <name>Task 1: Implement four metric aggregators with 14-day rolling-window semantics</name>
  <files>src/ccguard/server/services/metric_aggregators.py, tests/unit/test_metric_aggregators.py</files>
  <behavior>
    bash_calls_per_day_series:
      - Test: 5 ToolUseEvent rows with tool_name='Bash' across 3 distinct UTC days for machine m1 → returns 14-length list, the 3 days have correct counts, other 11 days are 0.
      - Test: events with tool_name='Edit' are excluded.
      - Test: events for a different machine_id are excluded.
      - Test: anchor day (today_utc) is the last element; oldest is today - 13 days.

    new_mcp_per_week_series (14-day series of rolling-week counts):
      - For each of the last 14 days (anchor_date d), count MCP server names that appear in the latest InventorySnapshot at or before d but did NOT appear in any InventorySnapshot strictly older than (d - 7 days).
      - Test: machine seeded with snapshots on d-10 (mcp_servers=[A,B]) and d-2 (mcp_servers=[A,B,C]) → at anchor=d, C is "new in last 7d" (count=1); at anchor=d-5, count=0; at anchor=d-1, count=1.
      - Test: empty inventory history → all-zero 14-length list.

    new_agents_per_week_series:
      - Same rolling-week semantics but the identity tuple is (AGENT_NAME_FIELD, AGENT_HASH_FIELD). A renamed agent with a new file_hash counts; an unchanged agent does not.
      - Test: agent "x" with file_hash=h1 in d-10 snapshot, file_hash=h2 in d-2 snapshot → at anchor=d, count=1 (because (x, h2) is new vs the older window).

    skill_dir_hash_changes_per_week_series:
      - For each anchor d, count skills whose dir_hash differs between the latest pre-d snapshot and the latest snapshot in the (d-7, d] window. If the latest pre-d snapshot is itself inside the window, count=0 (no change to compare against).
      - Test: skill "s" with dir_hash=h1 in d-10 snapshot, h2 in d-3 snapshot → at anchor=d, count=1.

    All four functions must accept (session: Session, machine_id: str, anchor_date: date | None = None) and return list[tuple[date, int]] of length 14. Default anchor_date = datetime.now(UTC).date().
  </behavior>
  <action>

      1. Computes the 14 anchor dates: `[anchor_date - timedelta(days=13-i) for i in range(14)]`.
      2. For bash_calls_per_day: one SQL query `SELECT date(ts) as day, COUNT(*) FROM tool_use_event WHERE machine_id = :m AND tool_name = 'Bash' AND ts >= :start GROUP BY day`. Pad missing days with 0.
