# Phase 2 — UI Design Contract: Anomaly Detection

> Visual and interaction contract for the Overview "Аномалии" card, the `/anomalies` table, and the `/anomalies/{machine_id}/{metric}` drill-down. Generated by gsd-ui-researcher.
>
> **Constraint:** Extends the same Russian-language admin UI (Jinja2 + HTMX + Tailwind CDN) as Phase 1. **Reuses Phase 1 design tokens verbatim** — no new colors, no new typography, no new font sizes. shadcn is **not applicable** (server-rendered Jinja, no React).

| Property | Value |

## Information Architecture

### Navigation addition

In `templates/base.html`, insert one new `<a>` between **Аудит** and **Политика**:

```html
<a href="/anomalies" class="block hover:bg-slate-800 px-3 py-2 rounded">Аномалии</a>
```

Final sidebar order:
`Обзор / Машины / Находки / Аудит / Аномалии / Политика / История / Настройки`

### Page structure: Overview enhancement

Append a new card below the existing fleet table in `templates/overview.html`:

```html
<div class="bg-white rounded-lg shadow p-4 mt-4"
     hx-get="/_partials/anomalies/overview"
     hx-trigger="every 60s">
  {% include "components/_anomalies_overview.html" %}
</div>
```

The partial renders a heading and the **top-5 most recent** anomaly findings (by `ts DESC`), or an empty state.

### Page structure: `/anomalies`

Top-down vertical stack inside `<main class="flex-1 p-8">`:

1. **Page heading** — `<h2 class="text-2xl font-semibold mb-6">Аномалии</h2>`
2. **Matrix card** — one card containing a single table: rows = machines, columns = the 4 metrics. Each cell shows a sparkline (14 daily/weekly points) + the most recent value. `class="bg-white rounded-lg shadow p-4 mb-4"`.

No filter bar in this phase (fleet is <100 machines; full table is acceptable). HTMX polling: every 60s on the wrapping card div, target `/_partials/anomalies/matrix`.

### Page structure: `/anomalies/{machine_id}/{metric}`

Top-down vertical stack:

1. **Page heading** — `<h2 class="text-2xl font-semibold mb-2">Аномалия — {{ machine_id[:12] }} / {{ metric }}</h2>`
2. **Breadcrumb / back link** — `<a href="/anomalies" class="text-sm text-slate-500 hover:underline">← Все аномалии</a>` (above the heading, `mb-4`)
3. **Baseline summary card** — small key/value strip: `median`, `σ`, `sample_count`, `baseline_ready`, `updated_at`. `class="bg-white rounded-lg shadow p-4 mb-4"`.
4. **Timeseries chart card** — 14-day CSS bar chart with baseline band overlay and outlier dots. `class="bg-white rounded-lg shadow p-4 mb-4"`.
5. **Findings list card** — recent `Finding` rows matching this `(machine_id, rule_id=anomaly.{metric})`, default LIMIT 50, columns: Когда / Значение / Отклонение / rule_id.

### URL contract

| Route | Returns |
|-------|---------|
| `GET /anomalies` | full `anomalies_feed.html` |
| `GET /anomalies/{machine_id}/{metric}` | full `anomaly_detail.html`. `metric` ∈ {`bash_calls_per_day`, `new_mcp_per_week`, `new_agents_per_week`, `skill_dir_hash_changes_per_week`} — 404 on unknown |
| `GET /_partials/anomalies/overview` | `components/_anomalies_overview.html` — top-5 recent findings with `rule_id LIKE 'anomaly.%'` |
| `GET /_partials/anomalies/matrix` | `components/_anomalies_matrix.html` — full machine×metric table with sparklines |

No query params on `/anomalies` in this phase. The matrix is the full fleet snapshot.

## Interaction & State

### Click-through

- **Overview card row** — each top-5 entry is a link `<a href="/anomalies/{{ machine_id }}/{{ metric }}" class="hover:underline">`. Whole row is clickable via wrapping `<a>` block (mirrors `findings_feed.html` row link pattern).
- **Matrix cell** — each sparkline cell is wrapped in `<a href="/anomalies/{{ machine.id }}/{{ metric }}" class="block hover:bg-slate-50 rounded">`. `machine_id` cell links to `/machines/{full_id}` (mirrors Phase 1).
- **Detail page** — read-only. No actions.

### HTMX polling

- Overview "Аномалии" card: `hx-trigger="every 60s"` (less critical than Phase 1 audit timeline @ 30s).
- `/anomalies` matrix card: `hx-trigger="every 60s"`.
- Detail page: **not** polled. Refresh requires user navigation.

### States

| State | Trigger | UI |
|-------|---------|-----|
| Default (with data) | findings exist + baseline ready for ≥1 machine | All cards populated |
| Empty — Overview card | no anomaly findings ever | `<p class="text-sm text-slate-400 py-6 text-center">Аномалий нет.</p>` |
| Empty — matrix | no machines reporting | `<tr><td colspan="5" class="py-6 text-center text-slate-400">Машин нет.</td></tr>` |
| Warm-up cell | `baseline_ready=False` for that `(machine, metric)` | Sparkline replaced with `<span class="text-xs text-slate-400">накопление…</span>`. No outlier coloring. |
| Empty — detail timeseries | <14 daily points OR `baseline_ready=False` | Chart card shows: `<p class="text-sm text-slate-400 py-6 text-center">Недостаточно данных для baseline (нужно ≥7 точек).</p>` Findings list still renders if any. |
| Loading | initial page render | Server-rendered HTML; HTMX swaps silently. |
| Error (server 5xx on partial) | HTMX poll fails | HTMX default: keep last successful render. No toast. |
| Outlier present | latest value > median + 3σ OR < median − 3σ | Sparkline last bar gets `bg-red-600` (overrides `bg-slate-700`); detail chart adds red dot at that x-position. |

### Confirmations / destructive actions

None. All three views are **read-only**.

### Sparkline spec (CSS bar chart, cell-sized)

A miniature horizontal row of **14 vertical bars** representing the last 14 daily (or 14 weekly) data points, oldest on the left.

**Container:**
```html
<a href="/anomalies/{{ machine.id }}/{{ metric }}"
   class="block hover:bg-slate-50 rounded p-1">
  <div class="flex items-end gap-px h-6 w-20" role="img"
       aria-label="Динамика {{ metric }} за 14 точек">
    {% for p in points %}
      <div class="flex-1 {% if loop.last and p.is_outlier %}bg-red-600{% else %}bg-slate-700{% endif %} rounded-sm"
           style="height: {{ p.height_pct }}%; min-height: {% if p.value > 0 %}1px{% else %}0{% endif %};"
           title="{{ p.label }} — {{ p.value }}"></div>
    {% endfor %}
  </div>
  <div class="text-xs font-mono text-slate-700 mt-1">
    {{ last_value }}{% if is_outlier %} <span class="text-red-600 font-semibold">выброс</span>{% endif %}
  </div>
</a>
```

**Sizing rules:**
- Container: **80px wide × 24px tall** (`w-20 h-6`)
- Bar `height` style: `(point.value / max_in_window) * 100%`
- Bar minimum height when `value > 0`: **1px**
- Gap between bars: **1px** (`gap-px`)
- Bar color: `bg-slate-700`, last bar `bg-red-600` if `is_outlier=True`
- Warm-up state: replace entire `<div class="flex...">` with `<span class="text-xs text-slate-400">накопление…</span>`

### Timeseries spec (detail page, CSS bar chart + baseline band)

A horizontal row of 14 vertical bars at full-card width, with a baseline band (median ± 3σ) overlaid.

**Container:**
```html
<div class="bg-white rounded-lg shadow p-4 mb-4">
  <h3 class="text-sm font-semibold text-slate-700 mb-3">Динамика 14 дней</h3>
  <div class="relative h-32">
    {# baseline band — absolute overlay #}
    <div class="absolute left-0 right-0 bg-slate-300 opacity-30 rounded-sm"
         style="bottom: {{ band_bottom_pct }}%; height: {{ band_height_pct }}%;"
         aria-hidden="true"></div>
    {# bars #}
    <div class="relative flex items-end gap-1 h-full" role="img"
         aria-label="Таймсериес метрики {{ metric }} с baseline-полосой">
      {% for p in points %}
        <div class="flex-1 relative">
          <div class="{% if p.is_outlier %}bg-red-600{% else %}bg-slate-700{% endif %} rounded-sm w-full"
               style="height: {{ p.height_pct }}%; min-height: {% if p.value > 0 %}2px{% else %}0{% endif %};"
               title="{{ p.label }} — {{ p.value }}"></div>
          {% if p.is_outlier %}
            <div class="absolute -top-1 left-1/2 -translate-x-1/2 w-2 h-2 bg-red-600 rounded-full"
                 aria-label="выброс"></div>
          {% endif %}
        </div>
      {% endfor %}
    </div>
  </div>
  <div class="flex justify-between text-xs text-slate-400 mt-2">
    <span>{{ points[0].label }}</span>
    <span>{{ points[-1].label }}</span>
  </div>
</div>
```

**Sizing rules:**
- Container height: **128px** (`h-32`) — same as Phase 1 audit timeline
- Bar minimum height when `value > 0`: **2px**
- Gap between bars: 4px (`gap-1`)
- Bar color: `bg-slate-700`, outlier bars `bg-red-600`
- Outlier dot: 8px circle (`w-2 h-2 bg-red-600 rounded-full`) positioned above the bar
- Baseline band: `bg-slate-300` at `opacity-30`, positioned absolutely; `band_bottom_pct = (median − 3σ) / max * 100`, `band_height_pct = (6σ) / max * 100` (clamped to [0, 100])
- If `baseline_ready=False`: omit the band, show empty-state paragraph instead of bars

### Baseline summary strip

<dl class="flex flex-wrap gap-x-6 gap-y-1 text-sm">

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Semantic structure | `<main>`, `<h2>` page title, `<h3>` card titles, `<table>` with `<thead>`/`<tbody>` for matrix, `<dl>` for baseline summary, `<ul>` for overview list |
| Chart accessibility | Each chart container has `role="img"` + Russian `aria-label`. Each bar has native `title` attribute exposing `{date} — {value}`. Baseline band has `aria-hidden="true"` (decorative). Outlier dots have `aria-label="выброс"`. |
| Color contrast | `bg-red-600` on white ≥ 4.5:1 ✓; `bg-slate-700` on white ≥ 4.5:1 ✓; baseline band at opacity 30% is decorative (data is encoded via bars + tooltip, not the band alone). |
| Focus states | Sparkline cell links use `focus-visible:ring-2 focus-visible:ring-slate-500 rounded`. Overview row links inherit Tailwind defaults. |
| Keyboard navigation | Tab order: back link → sparkline links left-to-right, top-to-bottom → findings rows. Native — no `tabindex` overrides. |
| Warm-up indication | Not color-only — text label `накопление…` accompanies the missing sparkline. |
| Outlier indication | Not color-only — text badge `выброс` + position dot + tooltip value. |

## Performance

| Concern | Decision |
|---------|----------|
| Matrix query | Single query joining `Machine` × `MachineBaseline` × last-14-points aggregate. For <100 machines × 4 metrics × 14 points = ≤5600 cells worth of data — well under SQLite envelope. |
| Sparkline data | Pre-computed by anomaly service when baseline updates; stored as JSON column on `MachineBaseline` (`recent_points` array of 14 floats) OR re-queried per request (planner decides). UI-SPEC requires only that the 14 numbers are available at render time. |
| HTMX polling | **60s** for Overview card and `/anomalies` matrix (less frequent than Phase 1 timeline @ 30s — anomalies don't change minute-to-minute). |
| Detail page | Server-rendered HTML, **not polled**. User refresh = full nav. |
| Payload size | Overview partial: 5 rows × ~150 bytes ≈ 1KB. Matrix partial: 100 machines × 4 cells × ~200 bytes ≈ 80KB worst case. Acceptable. |
| No new dependencies | **No chart library**. All charts = CSS + Jinja. Honors "CDN-light stack" constraint. |
| Tailwind CDN | Continues to use `cdn.tailwindcss.com` runtime — no build step. |
