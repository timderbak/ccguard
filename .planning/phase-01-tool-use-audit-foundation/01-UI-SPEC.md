# Phase 1 ‚Äî UI Design Contract: Tool-Use Audit

> Visual and interaction contract for the `/audit` page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Constraint:** This phase extends an existing Russian-language admin UI (Jinja2 + HTMX + Tailwind CDN). The contract below reuses existing tokens, components, and patterns ‚Äî no new design system is introduced. shadcn is **not applicable** (server-rendered Jinja, no React).


`–û–±–∑–æ—Ä / –ú–∞—à–∏–Ω—ã / –ù–∞—Ö–æ–¥–∫–∏ / –ê—É–¥–∏—Ç / –ü–æ–ª–∏—Ç–∏–∫–∞ / –ò—Å—Ç–æ—Ä–∏—è / –ù–∞—Å—Ç—Ä–æ–π–∫–∏`

### Page structure: `/audit`

Top-down vertical stack inside `<main class="flex-1 p-8">`:

1. **Page heading** ‚Äî `<h2 class="text-2xl font-semibold mb-6">–ê—É–¥–∏—Ç</h2>`
2. **Filter bar** (card 1) ‚Äî single `<form method="GET" action="/audit">`, mirrors `findings_feed.html` layout. Submission via native button = full page reload with URL params.
3. **Timeline chart** (card 2) ‚Äî 24h hourly buckets, CSS bar chart, HTMX-polled every 30s.
4. **Events table** (card 3) ‚Äî paginated list of recent events (default LIMIT 200, see Performance).

Each card: `class="bg-white rounded-lg shadow p-4 mb-4"`.

### URL contract

Filter form submits as GET to `/audit` with these query params (exact names ‚Äî backend must match):

| Param | Type | Allowed values | Default |
|-------|------|----------------|---------|
| `machine_id` | string (free text, fragment match) | any | empty (all) |
| `tool_name` | string (free text, exact match) | any (`Bash`, `Edit`, `Write`, `Read`, ...) | empty (all) |
| `decision` | enum | `allow`, `deny`, `error`, `` | empty (all) |
| `timeframe` | enum | `1h`, `24h`, `7d` | `24h` |

Empty/missing param = "no filter on this dimension". Backend treats `timeframe=24h` as the default when the param is missing.

### HTMX partial endpoints

Two new server-rendered chunk endpoints, following the existing `_partials/*` convention from `overview.html`:

| Endpoint | Returns | Polled by |
|----------|---------|-----------|
| `GET /_partials/audit/timeline` | `components/_audit_timeline.html` rendering of 24h hourly bucket data, honors current filter params (passed through) | timeline card every 30s |
| `GET /_partials/audit/events` | `components/_audit_events_table.html` rendering of current LIMIT 200 events matching filter params | (not polled in this phase ‚Äî only loaded on full page GET; left as a partial for future extraction) |

Note: only the **timeline** is HTMX-polled in this phase. The table is filtered via full page GET on filter form submit. This matches the existing pattern where time-sensitive aggregates poll and detail tables do not.

## Interaction & State

### Filter submission

- Submit button: `<button class="bg-slate-900 text-white text-sm rounded px-4 py-2">–§–∏–ª—å—Ç—Ä</button>`
- Pressing **Enter** in any text input submits the form (native browser behavior ‚Äî no JS).
- All currently selected filter values are preserved across submissions via `value="..."` / `selected` attributes (same pattern as `findings_feed.html`).
- "–°–±—Ä–æ—Å–∏—Ç—å" link: `<a href="/audit" class="text-sm text-slate-500 hover:underline self-center">–°–±—Ä–æ—Å–∏—Ç—å</a>` ‚Äî placed after the submit button. Resets all filters to defaults via plain navigation.

### HTMX polling

- Timeline card has `hx-get="/_partials/audit/timeline"` + `hx-trigger="every 30s"`.
- Polled endpoint receives current query string via `hx-include="closest form"` so polled data honors the active filter set.
- No client-side spinner needed (poll is silent/background).

### Click-through

- Each event row: `machine_id` cell is a link `<a href="/machines/{{ event.machine_id }}" class="hover:underline">{{ event.machine_id[:12] }}</a>` (mirrors `findings_feed.html`).
- No other cells are clickable in this phase.

### States

| State | Trigger | UI |
|-------|---------|-----|
| Default (with data) | events exist for filter | Filter bar + timeline + table populated |
| Empty (no data) | zero events match filter | Table shows single row: `<tr><td colspan="N" class="py-6 text-center text-slate-400">–ê—É–¥–∏—Ç-—Å–æ–±—ã—Ç–∏–π –Ω–µ—Ç.</td></tr>`. Timeline card shows: `<p class="text-sm text-slate-400 py-6 text-center">–ù–µ—Ç –¥–∞–Ω–Ω—ã—Ö –∑–∞ –≤—ã–±—Ä–∞–Ω–Ω—ã–π –ø–µ—Ä–∏–æ–¥.</p>` |
| Loading | initial page render | Server-rendered HTML ‚Äî no separate loading state. HTMX polled updates swap silently (no skeleton). |
| Error (server 5xx on partial) | HTMX poll fails | HTMX default: keep last successful render. No toast. (Acceptable for v0.2 admin tool.) |
| Filter-mismatch | filter combination yields empty | Same as Empty ‚Äî keep filter bar populated so user can adjust. |

### Confirmations / destructive actions

None in this phase. The `/audit` page is **read-only**. No delete, no edit, no policy mutation.

### Timeline chart spec (CSS bar chart, no JS library)

A horizontal row of 24 vertical bars representing hourly buckets for the last 24 hours, oldest on the left.

**Container:**
```html
<div class="bg-white rounded-lg shadow p-4 mb-4"
     hx-get="/_partials/audit/timeline"
     hx-trigger="every 30s"
     hx-include="closest form">
  <h3 class="text-sm font-semibold text-slate-700 mb-3">–ê–∫—Ç–∏–≤–Ω–æ—Å—Ç—å –∑–∞ 24 —á–∞—Å–∞</h3>
  {# server-rendered partial below #}
  <div class="flex items-end gap-1 h-32" role="img" aria-label="–ì–∏—Å—Ç–æ–≥—Ä–∞–º–º–∞ –∞—É–¥–∏—Ç-—Å–æ–±—ã—Ç–∏–π –ø–æ —á–∞—Å–∞–º">
    {% for bucket in buckets %}
      <div class="flex-1 bg-slate-700 rounded-sm"
           style="height: {{ bucket.height_pct }}%; min-height: {% if bucket.count > 0 %}2px{% else %}0{% endif %};"
           title="{{ bucket.hour_label }} ‚Äî {{ bucket.count }} —Å–æ–±—ã—Ç–∏–π"></div>
    {% endfor %}
  </div>
  <div class="flex justify-between text-xs text-slate-400 mt-2">
    <span>{{ buckets[0].hour_label }}</span>
    <span>{{ buckets[-1].hour_label }}</span>
  </div>
</div>
```

**Sizing rules:**
- Container height: **128px** (`h-32`)
- Bar `height` style: `(bucket.count / max_count_across_all_buckets) * 100%`
- Bar minimum height when `count > 0`: **2px** (visibility floor; see Spacing exceptions)
- Bar color: `bg-slate-700` (single color ‚Äî no decision split in v0.2; rationale: keep chart legible at this size; decision-split chart deferred to a future phase)
- Gap between bars: 4px (`gap-1`)
- Empty state: if all `count == 0`, replace chart with the empty-state paragraph (see States table).

**Hover tooltip:** native `title` attribute on each bar (no JS tooltip library).

**X-axis labels:** only the leftmost and rightmost hour labels shown (e.g. `15:00 25 –º–∞—è` ‚Ä¶ `14:00 26 –º–∞—è`) ‚Äî keeps the chart visually quiet.

## Performance

| Concern | Decision |
|---------|----------|
| Default row limit | **200 events** (`SELECT ... LIMIT 200`) ‚Äî enough to be useful, small enough that table renders in <50ms on SQLite |
| Pagination | Not in this phase. If `total > 200`, show footer row-count copy directing user to narrow filters. Real pagination deferred. |
| Timeline aggregation | Server-side `GROUP BY strftime('%Y-%m-%d %H', ts)` against `AuditRecord`. Index recommended on `(ts DESC)` (planner to confirm in DB schema). 24 rows back. |
| HTMX polling | **30s** for timeline only (matches `overview.html`). Table is **not** polled ‚Äî refresh requires user action (full nav). |
| Payload size | Timeline partial: ~24 buckets √ó ~50 bytes = <2KB. Table partial: 200 √ó ~200 bytes = ~40KB. Acceptable. |
| Initial page render | Server-rendered HTML in one request. No additional JS modules loaded beyond existing HTMX CDN (already in `base.html`). |
| No new dependencies | **No chart library** (no chart.js, no D3, no Apache ECharts). All chart logic is CSS + Jinja. Honors "CDN-light stack" project constraint. |
| Tailwind CDN | Continues to use `cdn.tailwindcss.com` runtime ‚Äî no build step added for this phase. |
| HTMX include scope | Timeline polls with `hx-include="closest form"` so it picks up filter values without re-fetching the whole page. |
