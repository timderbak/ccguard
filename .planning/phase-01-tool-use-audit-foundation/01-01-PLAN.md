<objective>
Build the two purest, side-effect-free agent-side primitives required for tool-use audit: the deterministic fingerprint function and the local SQLite WAL buffer. No hook entrypoint, no flusher, no settings.json wiring — those build on these in PLAN 02.

Purpose: Privacy-by-design (TUA-01) demands that raw `tool_input` never leaves the hook process. Fingerprinting is the privacy boundary. The buffer absorbs hook output without ever touching the network on the hot path.

Module `ccguard.agent.audit_hook.buffer`:
- `class ToolBufferDB(path: Path)` — context manager
  - `insert(*, ts: str, tool_name: str, fingerprint: str, decision: str, result_status: str) -> None`
  - `row_count() -> int`
  - `drain(limit: int = 200) -> list[BufferRow]` — returns oldest rows (FIFO), does NOT delete
  - `delete_ids(ids: list[int]) -> None` — used by flusher after successful POST
  - `trim_to_cap(cap: int = 10_000) -> int` — drop-oldest when over cap; returns rows deleted
- `BufferRow = TypedDict("BufferRow", {"id": int, "ts": str, "tool_name": str, "fingerprint": str, "decision": str, "result_status": str})`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tool_input → fingerprint | Raw tool_input crosses into hash function; MUST NOT escape |
| concurrent hook procs → buffer.db | Multiple Python processes write to the same SQLite file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | fingerprint.py | mitigate | Module docstring + code review: `tool_input` only flows into hashlib; never logged, never returned. Test `test_audit_fingerprint.py::test_no_input_leak_in_repr` asserts `repr(compute_fingerprint(...))` is just 16 hex chars. |
| T-01-02 | Tampering | buffer.db | mitigate | `BEGIN IMMEDIATE` + busy_timeout=5000 prevents lost writes under concurrent hooks (Pitfall #2 in RESEARCH). Multiprocessing test verifies. |
| T-01-03 | Denial of Service | buffer.db unbounded growth | mitigate | `trim_to_cap(10_000)` invoked from hook_main.py (PLAN 02) keeps disk usage bounded; tested directly here. |
| T-01-04 | Information Disclosure | buffer.db file mode | mitigate | PLAN 02 sets 0600 on parent dir via `default_config_dir`; this plan creates only the .db file which inherits perms. Document in module docstring. |
