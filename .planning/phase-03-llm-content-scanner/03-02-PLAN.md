<objective>
Build a single-purpose async wrapper around the official `anthropic` SDK that
scans one file's content and returns structured output via the `report_risk`
tool with `strict: true`. This is the only module that talks to Anthropic.

Purpose: Isolating the SDK call here (a) keeps secrets out of the service
layer, (b) makes mocking trivial in Plan 06 tests, and (c) lets the scan
service stay pure async coordination + DB writes.

- D-02: one-pass protocol (agent sends content+hash in one call; we don't ask)
- D-05: `strict: true` on the tool definition for reliable JSON
- D-06: Haiku 4.5 pricing $1/$5 per MTok (NOT the retired Haiku 3.5 rates)

Output: `LLMClient` class with `scan_content()` async method, declared dependency,
config plumbing for `ANTHROPIC_API_KEY`, and 6+ unit tests with the SDK mocked.

<interfaces>
<!-- Anthropic SDK contract for executor reference -->

Tool definition shape:
```
{
  "name": "report_risk",
  "description": "Report the risk classification of the scanned content.",
  "input_schema": {
    "type": "object",
    "properties": {
      "risk_score": {"type": "integer", "minimum": 0, "maximum": 100},
      "category": {"type": "string", "enum": ["jailbreak", "prompt-injection-template", "data-exfil", "privilege-escalation", "benign"]},
      "rationale": {"type": "string", "maxLength": 500}
    },
    "required": ["risk_score", "category", "rationale"]
  },
  "strict": true
}
```

Anthropic response: `response.content` is a list of content blocks; the tool_use block has `.type == "tool_use"`, `.name == "report_risk"`, `.input` is the structured dict. `response.usage.input_tokens` and `response.usage.output_tokens` give billing inputs.

Cost (cents) formula (D-06):
  cost_cents = round((input_tokens * 100 / 1_000_000) + (output_tokens * 500 / 1_000_000))
  i.e. $1/MTok input = 100 cents/MTok; $5/MTok output = 500 cents/MTok
</interfaces>
