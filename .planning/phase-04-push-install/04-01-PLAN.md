<objective>
Lay the schema foundation for Phase 4: extend the policy Pydantic v2 schema with
four new optional sections (`required_mcp_servers`, `required_skills`,
`required_agents`, `managed_claude_md_blocks`), add a new SQLModel table
`PolicyApplyEvent` for agent-reported apply outcomes, and register the table
with composite indexes via `create_all`.

module (plan 03), the CLI hook (plan 04), nor the audit page extension (plan
05) can store or read mandatory-section data. All four PUSH requirements
depend on this schema landing first.

  apply logic, but the Policy schema must allow MCP entries to round-trip the
  expected fields (name, command, args, env)

PolicyApplyEvent SQLModel + indexes, and two unit-test modules proving forward
+ backward compatibility and persistence.
</objective>
