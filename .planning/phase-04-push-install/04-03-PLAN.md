<objective>
Build the agent-side apply engine: an `atomic_io` helper, a `push_install`
module that snapshots the target files, applies the four mandatory sections
from the policy, and rolls back atomically on any error. Stdlib only.
POSIX-only (tempfile + os.replace) — no Windows special-case per project
constraints.

Purpose: This is the heart of PUSH-02 and PUSH-03. Plan 04 (CLI integration)
calls `apply()` and forwards its result to the audit endpoint; plan 06
verifies behavior end-to-end. Without this module the agent cannot perform
any centrally-managed configuration changes.

Locked decisions applied:
- D-2 snapshot scope = strictly target files (4 categories: skills, agents,
  ~/.claude.json, ~/CLAUDE.md); never whole-directory backups
- D-3 SKIP orphan deletion for v0.2 — do NOT write a managed-manifest.json
  and do NOT delete files that were previously managed but absent from the
  new policy; this is explicitly deferred to v0.3
- D-4 CLAUDE.md scope = user-home `~/CLAUDE.md` only; project-scope
  CLAUDE.md files are deferred to a later milestone
- D-5 `required_skills[].content` is the FULL file (frontmatter + body) —
  write the bytes as-is, no frontmatter assembly
- D-7 managed MCP entries are identified by `_managed_by: "ccguard"` —
  during merge, the agent REMOVES every existing entry in
  ~/.claude.json::mcpServers where `_managed_by == "ccguard"` (NOT by key
  prefix — safer per the locked decision), then ADDS the entries from
  `required_mcp_servers` with `_managed_by: "ccguard"` re-injected; user
  entries are left untouched

Output: Two new agent modules + 4 focused unit-test files covering the
atomic write, marker merge, MCP merge, and rollback paths.
</objective>
